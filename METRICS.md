# Метрики качества диаграмм

Используются в `utils/metrics.py` для автоматической оценки PlantUML-кода, сгенерированного многоагентной системой.

---

## Универсальные метрики

Вычисляются для **всех типов диаграмм**.

| Метрика | Название | Описание |
|---|---|---|
| `syntax_valid` | Синтаксис | 1 если код содержит `@startuml` и `@enduml`, иначе 0. Минимальная проверка корректности. |
| `diagram_lines` | Строк кода | Общее количество строк PlantUML-кода. Косвенная мера размера и детализации диаграммы. |
| `block_count` | Блоков | Количество именованных элементов диаграммы. Считается разными regex в зависимости от типа (см. ниже). |
| `relation_count` | Связей | Количество стрелок/связей между элементами. |
| `relation_variety` | Типов связей | Количество уникальных типов стрелок (`-->`, `*--`, `o--`, `..>` и др.). Отражает семантическое богатство диаграммы. |
| `avg_relations_per_block` | Связей/блок | `relation_count / block_count`. Средняя связанность элементов. Очень низкое значение (< 1) указывает на изолированные узлы. |
| `entity_coverage_pct` | Покрытие требований (%) | Доля значимых слов из текста требований (длиннее 3 символов, без стоп-слов), которые встречаются в PlantUML-коде. Bag-of-words метрика — приближённая оценка полноты. |
| `excess_elements_pct` | Лишних элементов (%) | Доля блоков диаграммы, названия которых **не встречаются** в тексте требований. Прокси для галлюцинаций модели — элементов, придуманных сверх задания. |
| `isolated_nodes_count` | Изолированных узлов | Количество блоков, имя которых не встречается ни в одной строке со стрелкой. Признак незаконченной диаграммы: сущность объявлена, но не подключена. |

## Метрики класс-диаграммы (`diagram_type = "class"`)

| Метрика | Название | Описание |
|---|---|---|
| `attribute_count` | Атрибутов | Количество строк вида `+ name : Type` — атрибутов с типами данных. |
| `method_count` | Методов | Количество строк вида `+ method()` — методов с сигнатурой. |
| `enum_count` | Enum | Количество объявлений `enum`. Наличие enum говорит о проработанности статусов и перечислений. |
| `interface_count` | Интерфейсов | Количество объявлений `interface`. |
| `package_count` | Пакетов | Количество объявлений `package`. Отражает группировку классов по подсистемам. |
| `has_notes` | Заметки | 1 если в коде есть хотя бы одна `note`. Признак документирования ключевых решений. |
| `avg_attributes_per_class` | Атр./класс | `attribute_count / block_count`. Средняя детализация классов. |
| `multiplicity_specified_pct` | Кратность связей (%) | Доля связей, для которых указана кратность (`"1"`, `"0..*"`, `"1..1"` и т.д.). Высокое значение — признак аккуратной проработки отношений. |

## Метрики диаграммы последовательности (`diagram_type = "sequence"`)

| Метрика | Название | Описание |
|---|---|---|
| `alt_block_count` | Ветвлений (alt) | Количество блоков `alt/else` — ветвлений в сценарии. Отражает покрытие альтернативных и исключительных сценариев. |
| `loop_count` | Циклов (loop) | Количество блоков `loop` — повторяющихся взаимодействий. |
| `activation_count` | Активаций | Количество операторов `activate` — явного указания времени жизни участника во время обработки. |

---

## Метрики компонентной диаграммы (`diagram_type = "component"`)

| Метрика | Название | Описание |
|---|---|---|
| `interface_count` | Интерфейсов | Количество явных интерфейсов между компонентами. |
| `package_count` | Пакетов (слоёв) | Количество `package` — архитектурных слоёв (Presentation, Business Logic, Data и т.д.). |

---

## Метрики диаграммы деятельности (`diagram_type = "activity"`)

| Метрика | Название | Описание |
|---|---|---|
| `swimlane_count` | Swimlane-полос | Количество уникальных swimlane-полос (`|Actor|`). Отражает число участников/подсистем в процессе. |
| `decision_count` | Решений (if) | Количество блоков `if/endif` — точек принятия решений. |
| `fork_count` | Параллелей (fork) | Количество блоков `fork` — параллельных ветвей выполнения. |

## Метаданные генерации

Хранятся в `meta.json`, не вычисляются из кода диаграммы.

| Поле | Описание |
|---|---|
| `generation_time_sec` | Время работы пайплайна агентов (в секундах). Зависит от модели и сложности входных данных. |

## Примечания

- **`entity_coverage_pct`** и **`excess_elements_pct`** — взаимодополняющие метрики. Первая показывает что из требований попало в диаграмму, вторая — что в диаграмме лишнее. В идеале: высокое покрытие + низкий процент лишних.
- **`isolated_nodes_count > 0`** — сигнал для ручной проверки: возможно, агент создал сущность, но забыл прописать связи.
- **`multiplicity_specified_pct`** применим только к class-диаграммам, так как кратность — специфика объектных отношений.
- Все метрики вычисляются статически через regex без исполнения кода — это быстро, но не идеально. Например, `entity_coverage_pct` не учитывает синонимы и переводы (AGV ≠ "робот").
