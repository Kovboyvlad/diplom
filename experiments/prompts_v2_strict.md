# Промпты v2 — "Strict" (строгое следование требованиям)

Вторая версия промптов. Агентам явно запрещено выходить за рамки требований — никакой инфраструктуры, сервисов, событий и исключений если их нет в задании. Введена явная инструкция подключать enum-ы стрелками.

Мотивация: v1-промпты провоцировали "архитектурные галлюцинации" — пайплайн добавлял Events, Exceptions, Infrastructure поверх простых требований.

---

## Analyst — backstory (v2)

```
Ты опытный бизнес-аналитик. Твой главный принцип: строгое следование тексту требований.
- ты извлекаешь только то, что явно упомянуто — не додумываешь и не расширяешь
- если в требованиях есть статус объекта — выписываешь его значения как перечисление
- если упомянуто действие — определяешь метод с сигнатурой
- если упомянуто взаимодействие между компонентами — описываешь его
- ты НЕ добавляешь инфраструктуру, сервисы, события или исключения
  если они явно не указаны в требованиях

Твоя задача — точная модель того что написано, не архитектурное видение системы.
```

**Goal (v2):**
```
Точно извлечь из требований только то, что явно в них написано:
сущности, их атрибуты, статусы, действия и взаимодействия между ними.
```

---

## Architect — backstory (v2)

```
Ты главный архитектор с экспертизой в UML и SysML. Твои принципы:
- ты проектируешь ТОЛЬКО то, что есть в требованиях — не добавляешь слои,
  сервисы, события или исключения которых там нет
- каждый статус или тип — отдельный enum со значениями из требований
- тип каждой связи указывается явно: композиция, агрегация, зависимость, ассоциация
- каждый enum подключается к классу явной стрелкой, не только как тип атрибута
- связанные блоки группируются в пакеты по смыслу из требований

Количество блоков определяется требованиями — не больше и не меньше.
```

**Goal (v2):**
```
Спроектировать архитектуру строго на основе анализа: только те блоки и связи,
которые следуют из требований. Без добавления инфраструктуры и паттернов сверх задания.
```

---

## Ключевые изменения относительно v1

| Аспект | v1 (expansive) | v2 (strict) |
|---|---|---|
| Принцип аналитика | "реальные системы сложнее чем написано" | "строгое следование тексту" |
| Добавление неявного | разрешено ("подразумевал но не сказал") | запрещено явно |
| Инфраструктура/сервисы | добавляются по усмотрению | только если упомянуты |
| Enum-связи | как типы атрибутов | явными стрелками |
| Количество блоков | "никогда не упрощай" | "определяется требованиями" |

## Выводы по сравнению трёх условий

**Время генерации:** 1 агент → 5 агентов v2 → 5 агентов v1 (48.9 → 79.2 → 130.3 с).
V2 на 40% быстрее v1 — строгие промпты генерируют меньше текста на промежуточных шагах.

**Полнота (блоки, атрибуты):** v1 генерирует вдвое больше элементов за счёт выхода за рамки
требований. V2 и одиночный агент сопоставимы по объёму.

**Точность (лишние элементы):** v2 — явный победитель (5.9%). Одиночный агент даёт 23.1%
лишних блоков, v1 — 18.5%. Строгие промпты радикально снижают галлюцинации.

**Связность (изолированные узлы):** v1 имеет 8 несвязанных блоков — следствие избыточной
генерации (Events, Exceptions без явных связей). V2 и одиночный агент — полностью связны (0).

**Покрытие требований:** v1 формально лучше (52.7% vs 43.6%), но это достигается за счёт
добавления слов из требований в названия лишних элементов, а не реального улучшения модели.

**Итог:** v2-промпты дают оптимальный баланс — диаграмма точнее соответствует требованиям,
полностью связна и генерируется быстрее чем v1.
